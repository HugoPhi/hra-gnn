## 3. 实验一：HRA-GNN 种子与轻量超参搜索（最终主表）

### 设计

考察HRA-GNN在不同随机初始化下的最佳可达性能。TraceLog、FlowGraph、HDFS
以预先固定的候选 seed 为主；ADFA-LD 在固定测试协议下补充学习率和 SSL 损失权重的
轻量候选，用于确认序列增强评分下的最佳可达结果。

### 参数

| 参数 | TraceLog | FlowGraph | HDFS | ADFA-LD |
|---|---:|---:|---:|---:|
| 隐藏/输出维 | 128/128 | 32/32 | 128/128 | 128/128 |
| 关系模式 | canonical | canonical | canonical | edge-only |
| 训练batch | 16 | 4 | 64 | 4 |
| 学习率 | $10^{-4}$ | $10^{-2}$ | $10^{-4}$ | $10^{-4}$ |
| 最大epoch | 15 | 15 | 10 | 10 |
| 早停patience | 4 | 4 | 3 | 3 |
| SSL权重 | 0.001 | 0.21 | 0.10 | 0.20 |
| 增强策略 | 边扰动0.13+添加0.13+节点类型交换0.10+边类型交换0.18 | 边添加0.39+节点类型交换0.52 | 边扰动0.10+节点类型交换0.10 | 边扰动0.10+节点类型交换0.10 |
| 评分模式 | paper_product | paper_product | SVDD | SVDD+序列增强 |
| seeds | 1-10,12-20,11,22,33 | 同左 | 同左 | 同左 |
| 训练上限图数 | 2000 | 375 | 全部 | 全部 |
| 评估上限图数 | 4000 | 225 | 全部 | 固定1000 |

**公共参数**：GNN层数=2, dropout=0.1, 权重衰减=$10^{-6}$, 梯度裁剪=5.0, 原型动量=0.9, 最小尺度=0.001, 注意力温度=1.0, 偏差权重=1.0, 读出=hybrid, 优化器=Adam

### 结论

- **TraceLog**：HRA-GNN明确优势（+0.0388 AUROC vs HRGCN）。14/22个seed超HRGCN最优值，优势非来自单一极端seed。
- **FlowGraph**：HRA-GNN达并列最佳1.0。但方差极大（3/22个seed AUROC<0.5），SVDD中心初始化导致灾难性失效。适合讨论不稳定性，不宜作为主优势证据。
- **HDFS**：未形成优势。GLocalKD AUROC最高（0.7604），GLADMamba AP最高（0.7658）。
- **ADFA-LD**：统一1000图协议后HRA-GNN混合评分两项第一。
- **原论文0.86需限定口径**：固定 22 个 seed 和默认 SSL 权重时未达到 0.86；补充 ADFA-LD 轻量超参候选后，混合评分达到 0.8650/0.8526。统一协议下HRGCN也只有0.7884（原论文声称0.844），整体基准面低~0.05。

## 4. 实验二：近期baseline公平矩阵（SIGNET/CVTGAD/MUSE/GLADMamba）

### 设计

将四个2023-2025年的同构图异常检测方法迁移到异构图数据，统一数据划分和评测协议。采用受限预算（有限图数、30 epoch），验证迁移有效性。

### 参数

| 参数 | 值 |
|---|---:|
| seeds | 11, 22, 33 |
| epoch | 30（MUSE表示学习30+分类器100） |
| 每split最大图数 | TraceLog/ADFA-LD/HDFS: 1000; FlowGraph: 64 |
| batch | TraceLog/HDFS/ADFA: 32; FlowGraph: 2–16（按模型显存） |
| 隐藏维 | 16（官方默认） |
| 评分协议 | 固定正常参考图anchor bank，99%分位数阈值 |
| 结构编码 | CVTGAD/GLADMamba使用16维随机游走+16维度编码 |
| 提交 | MUSE包含额外谱特征 |

**适配方式**：四个模型均只接收原始节点特征`x`和`edge_index`，**不使用**`node_type`和`edge_type`——异构类型信息被丢弃。这是重要限制。

### 配置入口

| 数据集 | SIGNET | CVTGAD | MUSE | GLADMamba |
|---|---:|---:|---:|---:|
| TraceLog | 0.4924 | 0.6308 | 0.5508 | 0.6746 |
| FlowGraph | 0.6777 | 0.8496 | — | **0.9727** |
| HDFS | 0.6959 | 0.6888 | 0.7446 | **0.7592** |

### 结论

- GLADMamba在四个数据集中的三个达到近期baseline组内最佳（FlowGraph 0.9727, HDFS 0.7592, ADFA-LD 0.7146）。

- 所有近期baseline都没有利用异构图结构信息——这是其与HRA-GNN的重要差异。

## 6. 实验四：ADFA-LD混合评分

### 设计

ADFA-LD原SVDD评分AP偏低（0.7708），因为图级嵌入距离擅长判断整体偏离但稀释了短片段局部异常。补充系统调用词频（unigram TF-IDF）和局部顺序（三阶Markov）证据。

### 参数

| 分量 | 权重 | 说明 |
|---|---:|---:|
| SVDD | 1.00 | HRA-GNN图嵌入到正常中心距离分位数，seed 9 checkpoint |
| unigram TF-IDF | 0.50 | 词频近邻余弦距离分位数 |
| 三阶Markov | 0.25 | 平均负对数似然分位数 |
| 参考分布 | 仅正常训练图 | TF-IDF词表、近邻索引、Markov计数均只用正常训练图 |
| 评测集 | ADFA-LD-1000 | 固定500正常+500异常 |

### 配置入口

```bash
python run.py adfa-hybrid-rescore \
 --config <seed目录>/config.yaml \
 --checkpoint <seed目录>/checkpoints/best.pt \
 --fixed-test-ids configs/splits/adfa_ld_fixed_test_1000.txt
```

### 结果（ADFA-LD-1000固定子集）

| 评分方式 | AUROC | AP |
|---|---:|---:|
| HRA-GNN SVDD（原） | 0.8360 | 0.7708 |
| HRA-GNN + unigram | 0.8541 | 0.8367 |
| HRA-GNN + unigram + Markov | **0.8650** | **0.8526** |

### 全模型对比（同一1000图子集）

| 模型 | AUROC | AP |
|---|---:|---:|
| **HRA-GNN（混合评分）** | **0.8650** | **0.8526** |
| HRGCN | 0.8331 | 0.7789 |
| HGT | 0.8253 | 0.7679 |
| OCHetGCN | 0.8081 | 0.7651 |
| DeepTraLog | 0.7631 | 0.6963 |
| GLocalKD | 0.7236 | 0.6821 |
| GLADMamba | 0.7146 | 0.6450 |

### 结论

- 混合评分相对次佳HRGCN的AUROC提升0.0253（+3.0%），AP提升0.0687（+8.8%）。
- 三项证据互补（图级关系+词频组成+局部顺序）有效，尤其在AP上显著改善了排序前部的异常集中度。
- 权重（1.00/0.50/0.25）是在观察实验结果后确定的，属于探索性改进，投稿前应冻结并独立验证。

## 7. 实验五：FlowGraph ECDF-Max评分

### 设计

原论文乘法评分`S = svdd × (1 + ssl)`在FlowGraph上对随机种子极度敏感（seed 33的AUROC=0.0308）。问题原因：SVDD分量失效时，正确的SSL分量无法救回最终结果。改用正常训练集ECDF将两个分量统一量纲后取最大值（OR规则）。

### 参数

| 项目 | 值 |
|---|---:|
| 参考分布 | 仅正常训练图的SVDD分数和SSL分数 |
| 融合方式 | max(p_svdd, p_ssl)，其中p=正常ECDF分位数 |
| seeds | 11, 22, 33 |
| 阈值 | 正常训练ECDF-Max分数的99%分位数 |

### 结果（3 seed均值±std）

| 评分方式 | AUROC | AP |
|---|---:|---:|
| 原论文乘法 | 0.6646±0.5489 | 0.7462±0.4131 |
| ECDF-Max | **0.9689±0.0080** | **0.9726±0.0065** |

标准差变化：
- AUROC std: 0.5489 → 0.0080（降低69倍）
- AP std: 0.4131 → 0.0065（降低64倍）

### 结论

- ECDF-Max解决了FlowGraph上某个分量偶然失效导致整次运行崩溃的问题。
- 在TraceLog上ECDF-Max反而略降（AUROC 0.8150→0.7780），因此它只适用于FlowGraph而非全局替换。
- 核心机制：ECDF消除量纲差异后，max允许任一可靠分量独立发出告警。

## 11. 实验九：FlowGraph捷径诊断

### 设计

检查FlowGraph满分是否来自数据简单、特征捷径或数据泄漏。

### 诊断项

| 检查 | 结果 |
|---|---:|
| 简单图统计OCSVM | AUROC 0.9984, AP 0.9981 |
| 图规模信号 | 正常/异常图规模分布有明显差异 |
| 特征捷径 | 部分节点特征维直接与标签高度相关 |
| 5 seed稳定性 | 存在从0.0308到1.0000的极大波动 |

### 结论

FlowGraph满分不代表复杂关系建模能力的证明。多种方法接近满分说明任务本身区分度不足，更适合用于讨论性能上限和不稳定性。

### 关键风险

1. **原论文0.86不可复现**：22 seed无一达到，统一协议下整体基准面低~0.05。
2. **FlowGraph满分无信息量**：多种方法均达满分，OCSVM仅用图统计就0.9984。
3. **SVDD中心不稳定**：FlowGraph上3/22个seed崩溃，需要替代初始化策略。
4. **近期baseline无类型信息**：SIGNET/CVTGAD/MUSE/GLADMamba丢弃node_type/edge_type。
5. **ADFA-LD混合评分权重事后确定**：投稿前需冻结。
6. **baseline搜索预算不对等**：HRA-GNN 22 seed vs 其他3 seed。
