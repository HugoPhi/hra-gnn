# 主表第一次实验

## 1. 实验目标

本轮依次运行 SIGNET、CVTGAD、MUSE 和 GLADMamba，检查四种近期图异常
检测方法迁移到 TraceLog、FlowGraph、HDFS-100k 和 ADFA-LD 后的有效性，
并把结果汇入第一版主表。MUSE 需要为每张图构造稠密邻接矩阵，因此不在
FlowGraph 上运行。

本文件是本轮实验的唯一进度台账。模型产生的原始 CSV、预测分数和指标 JSON
仍保存在 `artifacts/`，不提交大体积产物到 Git。

## 2. 主表统计口径

- 每个模型和数据集运行随机种子 `11、22、33`。
- 以测试集 AUROC 最高的完整运行为“最佳运行”。
- AP、F1、MCC 等指标必须取自该最佳 AUROC 运行，不能跨 seed 拼接。
- 原始三 seed 结果全部保留，以便审稿时追溯稳定性。
- 该口径会使用测试集结果选择 seed，存在乐观偏差。论文中必须明确写成
  “多次独立运行的最佳结果”，不能写成平均性能或稳定性能。

## 3. 第一次实验预算

| 数据集 | 每个 split 最大图数 | 说明 |
|---|---:|---|
| TraceLog | 1000 | 固定 seed 的分层随机子集 |
| FlowGraph | 64 | 图很大，CVTGAD/GLADMamba 节点计算开销高 |
| HDFS-100k | 1000 | 固定 seed 的分层随机子集 |
| ADFA-LD | 1000 | 训练集不足 1000 时使用全部训练图 |

SIGNET、CVTGAD 和 GLADMamba 训练 30 epoch。MUSE 的表示学习训练 30 epoch，
一类分类器训练 100 epoch。所有方法使用相同数据划分函数、正常训练参考分数和
无测试标签阈值；不同 seed 的分层子集会不同。

这是一轮受控预算实验，证据等级为 `first_main_table_not_final`，不是最终全量
论文结果。若某方法在该轮明显有效，再扩大到完整 split。

## 6. 最佳运行主表

最佳运行通过以下命令生成：

```bash
.venv/bin/python run.py table \
  --input artifacts/results/fair_matrix/main_table_first_signet/runs.csv \
  --input artifacts/results/fair_matrix/main_table_first_cvtgad/runs.csv \
  --input artifacts/results/fair_matrix/main_table_first_muse/runs.csv \
  --input artifacts/results/fair_matrix/main_table_first_gladmamba/runs.csv \
  --aggregation best \
  --selection-metric auc \
  --summary-csv artifacts/results/tables/main_table_first_best.csv \
  --output artifacts/results/tables/main_table_first_best.tex
```

已生成以下结果：

```text
artifacts/results/tables/main_table_first_recent_best.csv
artifacts/results/tables/main_table_first_recent_best.tex
artifacts/results/tables/main_table_first_all_models_best.csv
artifacts/results/tables/main_table_first_all_models_best.tex
```

`recent_best` 仅包含本轮四个近期方法。`all_models_best` 还合并了此前的
OCHetGCN、GLocalKD、HGT、DeepTraLog、HRGCN 和 HRA-GNN 等结果。

需要注意，旧的直接基线与本轮近期方法使用了不同的受控采样上限。因此
`all_models_best` 是用于决定后续实验优先级的第一版总表，不能直接作为论文最终
公平主表。最终主表必须让同一数据集上的全部模型共享完全相同的图 ID 和预算。

## 7. 异常与结论

### 7.1 可以进入下一轮的组合

- GLADMamba 在四个数据集上都是近期方法中 AUROC 最佳，值得优先扩大预算。
- MUSE 在 HDFS 上达到 AUROC 0.7446，且显存、时间都较低，值得跑完整 HDFS。
- MUSE 在 ADFA-LD 上可运行，但稠密邻接使用约 5GB 显存；图规模继续增大时不适用。
- CVTGAD 在 FlowGraph 上达到 AUROC 0.8496，但单 seed 训练约 16 分钟且使用
  10.4GB 显存，不宜盲目扩大图数。

### 7.2 不能只看最佳值的异常

- FlowGraph 上 SIGNET 的三个 AUROC 为 0.2754、0.1914、0.6777；GLADMamba
  为 0.4355、0.7842、0.9727，最佳值掩盖了很强的 seed 敏感性。
- SIGNET 和 CVTGAD 在 FlowGraph 上出现长时间 loss 平台或震荡。高 AUROC
  可能同时利用了该数据集明显的图规模/结构捷径。
- CVTGAD/FlowGraph 的最佳 AUROC 和 AP 分别为 0.8496 和 0.9055，但正常训练
  分数 99% 分位数阈值给出的 F1 为 0。这说明排序有效不等于阈值可直接部署。
- ADFA-LD 上四个方法的 AUROC 为 0.6367 至 0.7146，但 F1 都不超过 0.0571。
  后续必须单独研究正常校准分数漂移和阈值策略。
- CVTGAD/ADFA-LD 峰值显存为 14.1GB，16GB V100 上余量很小。

### 7.3 关于“最佳表现”口径

本轮按要求改为报告最佳运行，但测试 AUROC 参与了 seed 选择，所以结果存在选择
偏差。论文若采用该口径，应明确报告“从三个随机种子中选择测试 AUROC 最佳的一次
运行”，并在补充材料保留全部 seed。更严谨的替代方案是按无标签验证损失选 seed，
测试集只评估一次。
