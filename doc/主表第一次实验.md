# 主表第一次实验

## 1. 实验目标

本轮依次运行 SIGNET、CVTGAD、MUSE 和 GLADMamba，检查四种近期图异常
检测方法迁移到 TraceLog、FlowGraph、HDFS-100k 和 ADFA-LD 后的有效性，
并把结果汇入第一版主表。MUSE 需要为每张图构造稠密邻接矩阵，因此不在
FlowGraph 上运行。

本文件是本轮实验的唯一进度台账。模型产生的原始 CSV、预测分数和指标 JSON
仍保存在 `artifacts/`，不提交大体积产物到 Git。

## 2. 主表统计口径

- 每个模型和数据集运行随机种子 `11、22、33`。
- 以测试集 AUROC 最高的完整运行为“最佳运行”。
- AP、F1、MCC 等指标必须取自该最佳 AUROC 运行，不能跨 seed 拼接。
- 原始三 seed 结果全部保留，以便审稿时追溯稳定性。
- 该口径会使用测试集结果选择 seed，存在乐观偏差。论文中必须明确写成
  “多次独立运行的最佳结果”，不能写成平均性能或稳定性能。

## 3. 第一次实验预算

| 数据集 | 每个 split 最大图数 | 说明 |
|---|---:|---|
| TraceLog | 1000 | 固定 seed 的分层随机子集 |
| FlowGraph | 64 | 图很大，CVTGAD/GLADMamba 节点计算开销高 |
| HDFS-100k | 1000 | 固定 seed 的分层随机子集 |
| ADFA-LD | 1000 | 训练集不足 1000 时使用全部训练图 |

SIGNET、CVTGAD 和 GLADMamba 训练 30 epoch。MUSE 的表示学习训练 30 epoch，
一类分类器训练 100 epoch。所有方法使用相同数据划分函数、正常训练参考分数和
无测试标签阈值；不同 seed 的分层子集会不同。

这是一轮受控预算实验，证据等级为 `first_main_table_not_final`，不是最终全量
论文结果。若某方法在该轮明显有效，再扩大到完整 split。

```bash
CUDA_VISIBLE_DEVICES=0 .venv/bin/python run.py fair-matrix \
  --matrix configs/experiments/main_table_first_signet.yaml
```

`--force`。

## 6. 最佳运行主表

最佳运行通过以下命令生成：

```bash
.venv/bin/python run.py table \
  --input artifacts/results/fair_matrix/main_table_first_signet/runs.csv \
  --input artifacts/results/fair_matrix/main_table_first_cvtgad/runs.csv \
  --input artifacts/results/fair_matrix/main_table_first_muse/runs.csv \
  --input artifacts/results/fair_matrix/main_table_first_gladmamba/runs.csv \
  --aggregation best \
  --selection-metric auc \
  --summary-csv artifacts/results/tables/main_table_first_best.csv \
  --output artifacts/results/tables/main_table_first_best.tex
```

等待服务器实验完成后填写主表和最佳 seed。
