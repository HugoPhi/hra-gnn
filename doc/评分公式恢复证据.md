# MathType 评分公式恢复证据

## 背景

原始论文是旧版 `.doc` 文件，公式以 MathType OLE 对象存储。普通文本提取只能
得到 `EMBED Equation.DSMT4` 占位符，因此早期无法直接看到联合异常评分公式。

复现过程中使用 `olefile` 读取复合文档中的 `ObjectPool/*/Equation Native`
数据流，并检查 3.3 节附近的 MathType MTEF 记录。

提取带有 TeX 注释的对象可使用：

```bash
.venv/bin/python scripts/extract_mathtype_tex.py \
  ../paper/EC0260673_O_origin.doc
```

## 恢复结果

3.3 节对应的对象序列为：

```text
ObjectPool/_1234568018  S_dist(G_test) = ||g_test - c||_2^2
ObjectPool/_1234568020  S_ssl(G_test) = 1 - p_test
ObjectPool/_1234568021  S_anomaly(G_test)
                         = S_dist(G_test) * (1 + S_ssl(G_test))
```

最终两个复合公式没有保存完整 TeX 注释，因此根据 MTEF 字符和模板记录恢复。
对象 `_1234568021` 中依次出现：

```text
S_anomaly
=
S_dist
乘号
(
1
+
S_ssl
)
```

由此得到论文联合评分：

```text
S_anomaly(G) = S_dist(G) * (1 + S_ssl(G))
```

其中：

```text
S_dist(G) = ||g - c||_2^2
S_ssl(G) = 1 - p
```

## 代码对应

可执行配置：

```yaml
evaluation:
  score_mode: paper_product
```

实现位置：

```text
hra_gnn/model.py
HRAGNN.anomaly_score()
```

核心计算：

```python
svdd = (output.embedding - self.svdd_center).square().mean(dim=-1)
ssl_anomaly = 1.0 - output.ssl_logit.sigmoid()
return svdd * (1.0 + ssl_anomaly)
```

代码使用隐藏维均值而不是求和计算距离。这会对所有样本乘以相同的
`1 / output_dim`，不改变 AUC 和 AP 排序，但会改变绝对分数尺度。

## 为什么这一点重要

早期临时实现曾假设联合分数为加权和：

```text
S_dist + weight * S_ssl
```

当 SSL 分数方向在某些随机种子上发生反转时，加权和会主导排序并产生错误结果。
恢复论文乘法公式后，FlowGraph seed 22 的 AUC 从临时评分的 0.154 恢复为
0.9837。

早期临时评分结果只保留用于开发追踪，不能作为论文复现结论。
