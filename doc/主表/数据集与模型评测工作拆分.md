# 数据集与模型评测工作拆分

> 本文归档于 `doc/主表/`，记录主表数据适配和模型评测任务的完成状态。

更新时间：2026-07-02

## 一、总体交付目标

项目分为两个可以独立验收的工作包：

1. 数据集适配：统一支持 TraceLog、FlowGraph、HDFS、ADFA-LD；
2. 模型与评测：统一运行 HRA-GNN、HRGCN 和近期 baseline，计算多指标并生成 LaTeX 大表。

所有模型共享同一份图 ID 划分、输入特征和指标实现。任何模型不得用测试 AUC 选择 epoch。

## 二、工作包 A：数据集适配

### A1 TraceLog（已完成）

- 已支持原 HRGCN CSV 格式；
- 使用发布的 train/validation/test 图 ID；
- 读取节点特征、节点类型、边类型和图标签；
- 保持与论文现有实验兼容。

### A2 FlowGraph（已完成）

- 已支持发布的 train/evaluation 图 ID；
- 发布数据没有显式 `edge_type` 列，当前按单一边类型处理；
- 26 维事件编码属于节点特征，不能误写成 26 种边类型。

### A3 HDFS（真实子集验收完成）

输入：

- Loghub 的 `HDFS.log_structured.csv`；
- 官方 `anomaly_label.csv`。

构图：

- 一个 `block_id` 对应一张图；
- 每次日志事件出现对应一个节点；
- `EventId` 对应节点类型；
- 顺序、同模板前驱、同组件前驱对应三类边；
- 图标签使用官方 block 级标签。

已实现 `packed` 内存映射格式，避免训练时一次性把全部图复制进内存。

当前使用 Zenodo `HDFS_100k.log_structured.csv` 与 HDFS_v1 block 标签完成
可用性验收，共 7940 张图，其中 313 张异常图。该结果只能标记为 HDFS-100k，
最终论文对比仍需下载并处理 Loghub 完整 HDFS_v1。

### A4 ADFA-LD（完整真实数据验收完成）

构图：

- 一条 syscall trace 对应一张图；
- 一次 syscall 出现对应一个节点；
- syscall ID 对应节点类型；
- 顺序、同 syscall 前驱等关系构成边；
- 训练词表仅由正常训练轨迹建立，测试阶段未见 syscall 使用 unknown 类型。

划分保持官方目录语义：训练目录只用于训练，攻击目录只用于测试，正常 validation
轨迹确定性拆分为验证和测试正常样本。

完整数据共 5951 条轨迹：833 条正常训练、4372 条正常 validation 和
746 条攻击轨迹，与公开统计一致。

ADFA-LD 有 151 种训练词表 syscall。若使用
`源节点类型 × 边类型 × 目标节点类型`，会产生 68,403 种关系并在模型创建时
OOM。因此 ADFA-LD 使用 `relation_schema: edge_only`：

- 151 种 syscall 仍使用各自的输入投影；
- 消息传递参数只按顺序、同 syscall 前驱、同辅助标识前驱三类边共享；
- 关系参数量由节点类型平方增长改为只随边类型增长。

### A5 数据集验收标准

- 图数、正常/异常数与预处理报告一致；
- 所有 `edge_index` 均在节点范围内；
- 训练集不含异常；
- 四个数据集均可执行 `data-info` 和一次 smoke training；
- 同一预处理 seed 产生完全一致的图 ID 和划分；
- 转图配置、词表和样本来源可追溯。

## 三、工作包 B：模型实现与多指标评测

### B1 模型清单

第一层，论文和同源模型：

- HRA-GNN；
- HRGCN；
- OCHetGCN；
- HGT-one-class；
- DeepTraLog 风格图模型。

第二层，2023--2025 近期正式方法：

- SIGNET；
- CVTGAD；
- MUSE；
- GLADMamba。

官方仓库和 commit 固定在 `configs/baselines.lock.yaml`。除 SIGNET 外，其余已检查的近期
仓库没有明确 LICENSE 文件，因此不直接复制源码进入本仓库，而是通过脚本下载并记录
版本；发布前还需向作者确认代码再分发许可。

### B2 “实现正确”的三级验收

每个近期模型必须同时通过：

1. **结构验收：**关键模块、损失和异常分数与论文公式及官方代码一致；
2. **程序验收：**单图、batch、反向传播、checkpoint 恢复和固定种子测试通过；
3. **数值验收：**先在官方数据集和官方协议上复现合理区间，再进入四数据集公平协议。

不能只因代码能运行就标记为“复现正确”，也不能只因某次 AUC 接近论文就忽略数据泄漏。

### B3 官方协议与公平协议

每个方法保留两个结果：

- `*-official`：严格遵循官方仓库，用于验证实现；
- `*-fair`：使用本项目冻结划分、共同输入和无标签 checkpoint 选择，用于论文主表。

已审计发现：

- MUSE 官方代码用 validation AUROC 做 early stopping；
- GLADMamba 官方代码训练期间反复计算 test AUC，并按跨 trial 平均 test AUC 选择 epoch；
- CVTGAD 官方代码同样在训练期间评估 test AUC。
- CVTGAD 官方入口硬编码使用 GPU 1；本项目以独立 patch 改为尊重外部
  `CUDA_VISIBLE_DEVICES`，不改变模型、损失或评分逻辑。
- MUSE 官方入口固定执行 5 次试验、10 个表示检查点和 9 组分类器超参数；
  本项目 patch 仅把这些运行预算暴露为命令行参数，默认值保持官方行为不变。
- MUSE 使用了旧版 PyG 的 `torch_geometric.utils.subgraph` 模块路径；兼容 patch
  只修改导入位置，不改变 `subgraph` 函数及算法行为。
- CVTGAD/GLADMamba 的官方余弦相似度未处理零范数嵌入，在 FlowGraph 上会产生
  `NaN/Inf`。公平 runner 使用带 epsilon 的等价归一化和分母下限，并在结果中记录
  `numerically_stabilized=true`。

因此官方数字只能用于实现校验，不能直接进入公平主表。

### B4 多指标

统一输出：

- AUROC、AP；
- Precision@1%、Recall@1%；
- TPR@1%FPR；
- Precision、Recall、F1、MCC；
- 参数量、训练时间、推理时间、CPU/GPU 峰值内存；
- 五种子均值、标准差和每 seed 原始值。

F1/MCC 阈值使用正常训练分数的 99% 分位数，不读取测试标签。

### B5 LaTeX 大表

统一命令读取多个 suite 的 `runs.csv`，以模型为行、数据集和指标为多级列，生成：

- 汇总 CSV；
- `sidewaystable*` LaTeX 表；
- 每格五种子 `mean \pm std`。

表格依赖：

```latex
\usepackage{booktabs}
\usepackage{graphicx}
\usepackage{rotating}
```
