## 双 GPU 性能验证

优化后的实现已经在两张 RTX 4090 上验证：

- TraceLog 1000 图训练基准从 115.5 秒降至 11.2 秒。
- TraceLog 完整训练轮次处理 65,000 张图约需 360 秒。
- FlowGraph 完整训练轮次约需 25 秒。
- 普通实验使用批量预测。
- 昂贵的逐图关系诊断只对指定 checkpoint 开启。
- 实验套件根据 `metrics.json` 自动跳过已完成任务。

主要性能优化包括：

- 将一个 batch 中的图合成无跨图边的断连大图。
- 将逐关系 Python 循环改成按边批量变换和按“关系-目标节点”分段聚合。
- 将图级均值、最大值和 softmax 改成分段张量运算。
- 将 CSV 图数据转换为连续 NumPy 视图，避免每轮重复执行 DataFrame 切片。
- 将四种图增强中的逐元素操作改成候选池预计算和张量批量赋值。

这些优化不改变图之间的连接关系、关系编号、每图池化边界或损失定义。

### FlowGraph 单次结果

论文乘法评分下的五次 AUC：

```text
seed 11: 0.9836
seed 22: 0.9837
seed 33: 0.6414
seed 44: 0.9622
seed 55: 0.0308
```

FlowGraph 对随机种子极度敏感。该现象与论文使用“首批次均值初始化并固定 SVDD
中心”的策略相符：不同首批次会给出不同中心，而后续训练无法纠正中心偏移。

因此，不能只报告一次接近 1.0 的运行。修改稿至少应报告五随机种子的均值和
标准差，并分析失败种子。

## 尚不能声称“完全复现”的部分

- 五种子主模型已经完成，但全部基线、消融和参数扫描尚未全部执行到最终状态。
- HGT、DeepTraLog 和 GLocalKD 是统一协议下的适配实现，不是作者官方实现。
- FlowGraph 发布数据没有论文所称的 26 种显式边类型。
- 论文结果没有说明随机种子方差，当前实测无法稳定达到 FlowGraph 1.0。
- 关系原型的动量、初始化和异常污染鲁棒性仍需要专门实验。
- 首批次固定 SVDD 中心是明显的不稳定来源，需要增加替代初始化实验。
