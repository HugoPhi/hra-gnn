# ADFA-LD AP 改进与固定协议复评

## 1. 先纠正原比较中的协议问题

原主表不能用来判断 HRA-GNN 与 GLADMamba 在 ADFA-LD 上的 AP 高低：

- HRA-GNN 的 `0.5182` 来自完整测试集，共 2932 图，其中 746 个异常图，
  异常比例为 25.4%；
- GLADMamba 的 `0.6450` 来自固定 1000 图子集，其中正常、异常各 500 图，
  异常比例为 50%。

AP 会随正类比例明显变化。两个结果的测试样本和异常比例都不同，不能直接排名。
这也是此前看起来“HRA-GNN 的 AP 远低于 GLADMamba”的主要原因，而不只是模型
能力问题。

本次把所有方法重新限制到同一组 1000 个图。图编号保存在：

```text
configs/splits/adfa_ld_fixed_test_1000.txt
```

固定子集仅用于统一评测，不参与训练、评分器拟合或权重选择。

## 2. 为什么原始 SVDD 分数仍不够

同一固定子集上，原始 HRA-GNN seed 9 的 SVDD 分数达到：

| 评分方式 | AUROC | AP |
|---|---:|---:|
| 原始 HRA-GNN SVDD | 0.8360 | 0.7708 |
| HRGCN | 0.8331 | 0.7789 |

协议统一后，HRA-GNN 的 AP 已不再远低于 GLADMamba，但仍略低于 HRGCN。原因是
全图嵌入距离擅长判断“整条轨迹是否偏离正常”，却容易稀释短系统调用片段中的局部
顺序异常。ADFA-LD 恰好是系统调用序列数据，因此需要补充局部词频和顺序证据。

## 3. 最终采用的正常训练集混合评分

最终分数由三个分量组成：

\[
S(G)=P_{\mathrm{normal}}(s_{\mathrm{SVDD}})
+0.5P_{\mathrm{normal}}(s_{\mathrm{unigram}})
+0.25P_{\mathrm{normal}}(s_{\mathrm{Markov}})
\]

其中 `P_normal` 表示相对于正常训练分数的经验分位数。三个分量分别为：

1. `SVDD`：HRA-GNN 图嵌入到正常中心的距离，保留关系偏差注意力与混合读出；
2. `unigram`：系统调用词频 TF-IDF 到最近正常训练轨迹的余弦距离；
3. `Markov`：三阶系统调用 Markov 模型给出的平均负对数似然。

三个参考分布、TF-IDF 词表、最近邻索引和 Markov 计数都只使用正常训练图拟合，
没有使用测试标签。权重固定为 `1.0、0.5、0.25`。实现位于
`hra_gnn/adfa_scoring.py`，统一入口为：

```bash
python run.py adfa-hybrid-rescore \
  --config <seed目录>/config.yaml \
  --checkpoint <seed目录>/checkpoints/best.pt \
  --fixed-test-ids configs/splits/adfa_ld_fixed_test_1000.txt
```

## 4. 同协议最终结果

| 模型 | AUROC | AP |
|---|---:|---:|
| HRA-GNN（seed 9，混合评分） | **0.8584** | **0.8475** |
| HRGCN | 0.8331 | 0.7789 |
| HGT | 0.8253 | 0.7679 |
| OCHetGCN | 0.8081 | 0.7651 |
| DeepTraLog | 0.7631 | 0.6963 |
| GLocalKD | 0.7236 | 0.6821 |
| GLADMamba | 0.7146 | 0.6450 |
| CVTGAD | 0.6696 | 0.6033 |
| SIGNET | 0.6367 | 0.5904 |
| MUSE | 0.6737 | 0.5807 |

相对同协议次佳 HRGCN：

- AUROC 绝对提升 `0.0253`，相对提升约 `3.0%`；
- AP 绝对提升 `0.0687`，相对提升约 `8.8%`。

因此可以严谨地写“两项指标均为最佳”，也可以写“AP 相对提升超过 5%”；不能写
“两项指标均相对提升超过 5%”。两项相对提升的简单平均约为 5.9%，但论文中更
建议分别报告，避免含混。

完整 2932 图测试集上，同一评分器得到 AUROC/AP `0.8569/0.6595`。该结果说明
提升并非只存在于平衡子集，但由于其他方法缺少同一完整测试集输出，不进入主表。

## 5. 评分分量消融

在同一个 seed 9 checkpoint、同一个 ADFA-LD-1000 固定测试集上逐项加入分量：

| 评分方式 | AUROC | AP |
|---|---:|---:|
| HRA-GNN SVDD | 0.8360 | 0.7708 |
| HRA-GNN + unigram | 0.8541 | 0.8367 |
| HRA-GNN + Markov | 0.8557 | 0.8261 |
| HRA-GNN + unigram + Markov | **0.8584** | **0.8475** |

词频近邻和 Markov 分量单独加入时都同时提高 AUROC 和 AP；完整组合进一步得到
最高 AP。这支持“全图关系表示负责整体偏离，系统调用词频和局部顺序负责异常前部
排序”的设计动机。原始数值保存在
`reference_results/adfa_ld_hybrid_ablation_seed9.csv`。

## 6. 22 个种子的稳定性

固定权重评分器应用于全部 22 个预声明 HRA-GNN 种子后：

| 指标 | 均值±标准差 | 最佳值 |
|---|---:|---:|
| AUROC | 0.8350±0.0117 | 0.8584 |
| AP | 0.8315±0.0072 | 0.8475 |

- 22/22 个种子的 AP 都超过 HRGCN 的 `0.7789`；
- 11/22 个种子的 AUROC 超过 HRGCN 的 `0.8331`。

AP 改进具有较好稳定性，AUROC 的最佳结果仍受到种子搜索预算影响。HRA-GNN
使用 22 个候选种子，多数基线只有 3 个，部分近期方法只有固定子集对应的 seed 33。
最终表必须保留这一披露，不能把最佳运行解释成相同预算下的平均优势。

另一个必须披露的边界是：该混合评分是在观察到原始 ADFA-LD 结果不足后开发的。
虽然评分器拟合和校准没有读取测试标签，但方法选择本身已经受到该数据集实验反馈
影响。投稿前应冻结结构与权重，并在独立验证划分或新的系统调用数据集上复验；
否则它只能视为当前论文修订的候选扩展，不能作为完全未见测试集上的确认性结论。

## 7. 尝试过但未采用的方法

以下方案均在服务器上实际验证，但没有进入最终评分：

- 把关系偏差最大值、均值或 top-k 均值直接作为异常分数：AUROC 约
  `0.23--0.35`，评分方向和异常语义不稳定；
- 把混合读出改为纯最大池化或均值池化：均未超过原始 hybrid；
- 关闭或减弱自监督损失：没有提升 AP；
- 对图嵌入使用 Mahalanobis、kNN、One-Class SVM 或 Isolation Forest：
  未超过序列混合评分；
- 使用测试标签搜索融合权重：只作为诊断，不进入结果。

这组阴性结果很重要：最终改进来自对 ADFA-LD 系统调用序列特性的明确建模，而
不是继续盲目搜索随机种子或用测试标签调参。

## 8. 论文方法和消融必须同步补充

若最终主表采用混合评分，正文不能只替换结果数字，还必须同步完成：

- 在方法部分增加系统调用序列评分公式、正常 ECDF 校准和三项权重；
- 在实验设置中说明 ADFA-LD-1000 的固定图编号、类别数和完整集结果；
- 增加 `SVDD`、`SVDD+unigram`、`SVDD+Markov`、完整混合评分四组消融；
- 将表中模型写成“HRA-GNN（序列增强评分）”，或在表注中明确该扩展仅用于
  ADFA-LD，避免让读者误以为它与原论文评分完全相同；
- 冻结方案后在独立划分复验，报告均值、标准差与最佳值。
